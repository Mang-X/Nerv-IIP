# MAN-528 Finished-Goods Inventory Link Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose a stable `RequestNo + WorkOrderId + ProducedLotNo` path from a MES finished-goods receipt to the exact Inventory movements and current lot balances, including explicit not-posted, failed, partial, quality-restricted, and posted states.

**Architecture:** Inventory remains the authority for movements and current balances. A new Inventory read endpoint resolves immutable movement provenance by `sourceService + sourceDocumentId/sourceDocumentLineId`, then projects the matching current ledgers without guessing by SKU or similar lot. BusinessGateway composes the exact MES receipt with that Inventory response and enforces both MES receipt-read and Inventory ledger-read permissions.

**Tech Stack:** .NET 10, FastEndpoints, MediatR, EF Core 10, PostgreSQL, CAP Redis Streams, NSwag/FastEndpoints OpenAPI, Hey API, xUnit.

## Global Constraints

- Base commit is exactly `cd15fe54d4bbfb98a311823ddd5b873bf3b0ec5b`.
- Scope is only Linear MAN-528 / GitHub #972; do not implement MAN-518's sales-order timeline UI.
- Never read another service's schema. BusinessGateway may compose only public service HTTP contracts.
- Inventory owns balances; MES and BusinessGateway must not persist copied stock facts.
- A missing source match returns an explicit state and never falls back to SKU, similar lot, or fuzzy identifiers.
- Any new business-service endpoint is `exposed`, has a same-PR BusinessGateway facade, and updates `facade-coverage-matrix.json`.
- OpenAPI snapshots and generated clients are regenerated by governed commands, never hand-edited.
- Production code follows test-first red-green-refactor; generated artifacts are exempt from hand-written TDD.

- [ ] **Step 3: Implement the aggregate state machine**

Use exact status precedence:

```text
InventoryPostingFailed + no movement -> postingFailed

otherwise                            -> posted
```
