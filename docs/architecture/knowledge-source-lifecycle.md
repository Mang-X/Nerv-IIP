# 知识源生命周期说明

本文档补齐 ADR 0005 中提出的知识源生命周期细节，定义 Knowledge 服务在首批实现中如何处理知识源注册、首次导入、增量更新、重建索引、停用和删除同步。本文档只冻结服务边界和行为口径，不决定具体嵌入模型、分块参数或后台 UI。

## 目标

1. 让 Knowledge 的 ingestion 行为具备可追踪、可重试和可审计的生命周期。
2. 保证检索结果始终携带权限上下文和可回溯引用。
3. 避免 AI Integration、Gateway 或前端绕过 Knowledge 自行维护向量索引。
4. 为后续接入 OCR、表格解析、多模态处理保留扩展空间。

## 核心对象

| 对象 | 职责 | 首批说明 |
| --- | --- | --- |
| KnowledgeSource | 管理知识来源、权限边界、同步策略与生命周期状态 | 可以表示 File Storage 文件集合、受控外部来源或后续外部系统来源。 |
| SourceDocument | 表示知识源下的单个原始文档 | 保存来源标识、版本指纹、解析状态和引用元数据。 |
| IngestionJob | 承载首次导入、增量同步、重建索引、删除同步等异步任务 | 需要记录触发原因、开始结束时间、失败原因和重试信息。 |
| DocumentChunk | 文档抽取和分块后的检索单元 | 需要保留源文档引用、位置、权限标签和内容指纹。 |
| EmbeddingRecord | 分块对应的嵌入结果 | 与向量索引条目关联，支持重建和失效。 |
| RetrievalCitation | 检索返回的引用 | 至少能回溯到知识源、源文档、片段位置和权限上下文。 |

## 生命周期状态

### KnowledgeSource

| 状态 | 含义 | 允许的后续动作 |
| --- | --- | --- |
| Draft | 已创建但尚未进入检索服务 | 配置权限、配置来源、触发首次导入、归档。 |
| Active | 可被同步、检索和引用 | 增量同步、重建索引、暂停、归档。 |
| Paused | 暂停同步和检索曝光 | 恢复为 Active、重建索引、归档。 |
| Rebuilding | 正在重建索引 | 完成后回到 Active 或 Paused；失败时保留旧索引可用性策略。 |
| Archived | 已归档，不再参与检索 | 保留审计和引用历史；可按策略执行物理清理。 |

### SourceDocument

| 状态 | 含义 |
| --- | --- |
| Pending | 已发现但尚未处理。 |
| Extracting | 正在抽取文本和结构化内容。 |
| Chunking | 正在分块和生成检索单元。 |
| Embedding | 正在生成嵌入向量。 |
| Indexed | 已写入索引并可被检索。 |
| Failed | 当前版本处理失败，需要记录失败原因。 |
| Stale | 来源已变化，当前索引等待更新或失效。 |
| Deleted | 来源已删除，索引和引用需按策略失效。 |

## 生命周期流程

### 1. 知识源注册

1. 用户或平台服务创建 KnowledgeSource，携带组织、环境、来源类型、权限范围和同步策略。
2. Knowledge 校验来源可访问性，但不在注册事务中执行大规模文档处理。
3. 注册成功后，知识源进入 Draft 或 Active；若选择立即导入，则创建首次 IngestionJob。

### 2. 首次导入

1. IngestionJob 枚举 SourceDocument。
2. 对每个文档执行抽取、分块、嵌入和索引写入。
3. 每个 DocumentChunk 必须携带来源引用、权限标签和内容指纹。
4. 部分文档失败不应阻断整个知识源的其他文档完成索引，但任务结果必须暴露成功数、失败数和失败原因。

### 3. 增量更新

1. Knowledge 根据同步策略或手动触发检查来源变化。
2. 新增文档进入 Pending，变更文档标记为 Stale 并重新处理。
3. 未变化文档复用现有索引和引用信息。
4. 删除文档不立即静默消失，应进入 Deleted 或失效流程，确保审计和引用历史可解释。

### 4. 重建索引

1. 当分块策略、嵌入模型、向量库配置或权限规则变化时，创建 Rebuild 类型 IngestionJob。
2. 重建期间不要求停止旧索引服务；首批实现可以采用“旧索引继续可用，新索引完成后切换”的策略。
3. 若重建失败，必须保留失败原因，并明确旧索引是否仍可检索。

### 5. 暂停与归档

1. Paused 知识源不再对检索请求曝光，也不执行自动同步。
2. Archived 知识源不再参与检索；原始文件、分块、嵌入和引用历史按保留策略处理。
3. 归档不等于立即物理删除，尤其不能破坏已经产生的审计记录与历史引用解释。

## 权限与引用规则

1. KnowledgeSource、SourceDocument 和 DocumentChunk 都必须携带组织与环境上下文。
2. 检索请求必须经过权限过滤后再返回内容片段。
3. 检索结果必须返回 RetrievalCitation，不能只返回脱离来源的纯文本。
4. AI Integration、Gateway 和平台服务只能消费 Knowledge 的检索接口，不应绕过权限过滤直接访问 Qdrant、File Storage 或对象存储。
5. Knowledge 引用原始文件时应使用 File Storage 的 fileId 或短期下载授权，不应暴露内部存储密钥、objectKey、临时访问凭证或未授权路径。

## 失败与重试

1. IngestionJob 需要记录失败阶段，例如 source-access、extract、chunk、embed、index、permission-sync。
2. 失败原因需要区分可重试与不可重试。
3. 对单文档失败，优先允许任务部分成功，并在结果中列出失败文档。
4. 对来源不可访问、权限配置错误等全局失败，应停止任务并保留可诊断信息。
5. 重试不应重复创建已成功的索引条目，应通过来源指纹、文档指纹和分块指纹控制幂等。

## 首批非目标

1. 不冻结 OCR、表格解析、多模态解析供应商。
2. 不冻结嵌入模型、rerank 策略、chunk size 和 overlap 的最终调优值。
3. 不实现跨知识源的复杂版本比较界面。
4. 不允许任何服务以“临时查询”为理由绕过 Knowledge 权限过滤直接查向量库。

## 首批验收标准

1. 可以创建 KnowledgeSource，并触发一次首次导入任务。
2. IngestionJob 能记录成功、失败、重试和耗时信息。
3. Indexed 文档可通过 Knowledge 检索接口返回内容片段和引用。
4. 检索结果受组织、环境和权限上下文过滤。
5. 归档或暂停后的知识源不再出现在普通检索结果中。
