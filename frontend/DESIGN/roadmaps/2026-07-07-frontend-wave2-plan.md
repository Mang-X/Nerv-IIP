- **DLQ 运维、IAM 用户禁用/密码策略** → 属于 `apps/console`（平台控制台，IAM Admin Console 已在此），**不是** business-console。business-console 走查报告的"无处安放"结论据此修正。

- **采集连接器/点位映射管理** → 待裁定：#683 连接器的配置面是配置文件/AppHub 侧还是 API？若是 Connector Host 配置文件驱动，则短期不做 CRUD 页面，只做**只读状态页**（连接器在线状态/采集健康，数据源 AppHub 心跳+状态快照）。需核实后端事实再定。
- **设备控制下发** → business-console 设备详情页 ✅；**PDA 明确不做**（误触风险，走查建议采纳：若未来必须上移动端，放高权限模式+工号复核）。

### X3. Business Console 导航演进
顶部域已 12+ 个，接近过载。建议：新增「平台管理」域仅在 console app（见 X2）；business-console 侧把"通知偏好"挂用户菜单而非新域；导航项继续遵守权限码绑定与图标规范。

## 7. 专项 A：交互体验系统性重设计（PC + 移动）

**方法论**（不是一次性大爆炸，而是"标准先行、逐域落地、实机验证"）：
1. **A1 交互模式规范 v1**：扩展 `DESIGN/patterns/`（已有 list-workbench/master-data-templates 先例），新增/修订：
   - **表单承载分级**：≤3 字段快捷操作 → Dialog；4~8 字段或含动态行 → Sheet 侧滑；复杂多段/多步 → 独立页。当前"一切皆 Dialog"是最大痛点。
   - **行操作分级**：每行最多 1~2 个高频动作提为行内按钮（报工/确认/开始检验），其余收 DropdownMenu；破坏性动作永远二次确认+原因。
   - **列表-详情模式统一**：轻详情用抽屉（保列表上下文），重对象（工单/NCR/ECO）用独立页；同域内不混用。
   - **操作后引导**：成功态必答"下一步去哪"（继续创建/查看详情/返回列表三选）；跨域写操作后关联查询失效刷新。
   - **空态带 CTA**：不只文案，给"去创建/去配置"行动入口。
   - **批量模式**：复选框+批量动作栏的统一形态（报警批量确认先用）。
   - **筛选状态保持**：列表筛选进 URL query，跨页返回不丢。
2. **A2 实机全景 UX 走查**：起整栈逐域点查（沿用 nerv.ps1 dev + seed 流程），按 A1 标准打分记录，产出逐域问题清单——这一步做完才有资格说"逐个设计"，代码走查看不出真实流畅度。
3. **A3 逐域重设计**：与功能批次（C1~C5）合并执行——每个功能 issue 的交互方案先由我出**文字交互稿**（页面结构/操作流/状态反馈逐条列出）经确认再实施，不让 codex 自由发挥。

**已知问题初稿**（代码走查可见，待 A2 实机确认补全）：创建/编辑全用 Dialog（完工入库 4 字段+NCR 处置多段尤其不适）；高频操作藏 Dropdown 发现性差；列表-详情抽屉/独立页混用；返回丢筛选；成功后无引导；批量缺失；PDA 数值输入未用 NumberKeyboard、多字段单 Sheet、部分流程无步骤提示。

## 8. 专项 B：组件库品牌化 + 四场景文档重构

**核实后的体系事实**：base（shadcn 原版，零改动）+ 四场景层——PC `pc/`（35 目录 95+ 件，Pro 后缀）、mobile（独立 `ui-mobile` 包 43 件，Mobile 前缀+裸名混杂）、screen（28+ 件，Screen 前缀）、touch（5 件，平板工位）。命名三种风格并存；console/screen app 无合约测试守护；docs 站 desktop 45 页 vs mobile/screen 明显偏科；token 文档不分场景（`--nv-scr-*` 等场景差异只在组件层体现）。

> **已决策（2026-07-07）**：品牌前缀 `Nv`，组件库品牌名 **NvUI**（包名方案在 ADR 定：`@nerv-iip/nvui` / 保持 `@nerv-iip/ui` 仅改组件名，倾向前者）；**迁移在业务批（C/P）之前完成**；token 允许跨场景取值相同（设计理念一致，如动效统一 motion-v 封装），但**名称必须按场景命名空间隔离**（primitive 层共享，semantic 层按场景前缀：PC `--nv-*`、screen `--nv-scr-*`（现 `--nv-scr-*` 迁移）、mobile `--nv-m-*`、touch `--nv-t-*`，具体在 ADR 冻结）；**样式隔离制度化**：全部组件样式进 CSS cascade layer，解决 VitePress 文档站 `--vp-*` 覆盖/revert-layer 嵌入坑，文档站嵌入规范写入 governance。

**B1 命名规范 ADR（先决策再动手）**——品牌前缀候选：

- **前缀具体用词（Nv/Nerv/其他）由你定**，ADR 同时冻结：shadcn 原版继续零改动零重命名（governance 既有红线）、新组件命名规则、场景归属判定规则。
