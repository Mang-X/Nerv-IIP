# PC / 桌面控制台组件层 · 产品定位（product.md）

> `@nerv-iip/ui` 的 `pc/` `blocks/` `layout/` 层 —— console（平台管理台）与
> business-console（业务操作台）的组件库。AI 编码与人工开发**新建或改动本层组件、
> 设计新页面前先读本文件**。参数细则见 `frontend/DESIGN/foundation.md`，
> 交互验收口径见 `frontend/DESIGN/patterns/interaction-patterns.md`。

## 一句话定位

给每天 8 小时高频操作的工厂计划员、主管、检验员用的信息密集操作台 ——
冷静、快、可预期，且经得起细看。

## 受众与场景

- **计划员 / 调度**：排产、派工、报工审核 —— 高频录入 + 大表格扫读，鼠标键盘并用。
- **值班主管**：跨域监控 + 审批 —— 状态语义和异常必须一眼可辨。
- **质检 / 仓管 / 采购**：单据流转 —— 惯例一致性比个性更重要。
- **环境**：办公室/产线办公区，标准显示器，长时间连续使用，常开多标签。

## 气质：三层兼顾（owner 裁决 2026-07-16）

专业、产品感、B端熟悉感不是三选一，分层各管一件事：

| 层                              | 取向         | 意味着                                                                                  |
| ------------------------------- | ------------ | --------------------------------------------------------------------------------------- |
| **骨架**（信息架构 / 操作范式） | 成熟 B端惯例 | 对标 SAP / D365 / Ant 的成熟操作流；同类操作同长相同位置；**可预期优先于独创**          |
| **基调**（色彩 / 密度 / 排版）  | 冷静工业     | 近黑 `--primary`、中性铬、**高密度为默认档**（text-sm + 紧凑行高）、克制装饰            |
| **触感**（微交互 / 细节）       | 产品级精致   | 动效令牌统一手感、hover/focus/active/loading 状态完备、细节经得起放大 —— 专业不等于粗糙 |

## 设计原则

1. **信息密度优先** —— 高密度是对专家用户的尊重，一屏多一行就少一次翻页。
   密度靠排版纪律获得（`foundation.md` Typography / Data Typography），
   不靠缩小字号到不可读。
2. **可预期** —— 规范没覆盖时先问"成熟 B端产品怎么做"，再问"我们要不要不同"。
   同一事实同一数据源同一呈现。
3. **品牌蓝主 CTA** —— 页面主操作按钮**常规使用 `variant="brand"`**（动态品牌色，
   每页/每工具栏唯一）；对话框/抽屉内的确认主操作用近黑 `default`；
   其余层级 `outline` / `ghost`。brand 同时仍是链接、焦点、图表、选中态的强调色。
4. **键盘基本可达** —— 表单 Enter 提交、Esc 关闭浮层、Tab 顺序合理、
   `focus-visible` 完备。这是验收项；全局快捷键体系（NvCommand 命令面板）
   是后续迭代方向，不做本层门槛。
5. **状态即语义** —— 状态一律 `NvStatusBadge`/`NvStatusDot` 五 tone；
   数字排版走 Data Typography 口径（右对齐、tabular-nums、`—` 空值）。

## 动效契约

与全系统同源：`--nv-ease-*` / `--nv-duration-*` 令牌，press 收缩不回弹，
全部降级 `prefers-reduced-motion`。清单：`frontend/DESIGN/motion-interaction.md`。

## Do / Don't

**Don't**

- ❌ 别把接口平铺成页面 —— 先想"这页给谁用、主操作是什么"（interaction-patterns）。
- ❌ 别放机械 KPI（本页 X 行）—— 只放帮助行动的语义指标。
- ❌ 别加无状态含义的装饰动效 / 发光 / 渐变文字。
- ❌ 别用 raw palette 类名 / 裸 hex —— 语义令牌。
- ❌ 别和组件搏斗 —— 凑合要 2+ 个补丁 props 就按选件阶梯新建（`governance.md`）。

**Do**

- ✅ 用真实业务数据的形状设计（`WO-` 单号、真实数量级、真实产线名）。
- ✅ 空态 / 失败态 / 无权限态给出下一步动作。
- ✅ 文案说业务的语言（`DESIGN/index.md` Copy Rule）。
- ✅ 状态完备：default / hover / focus-visible / active / disabled / loading / selected。
