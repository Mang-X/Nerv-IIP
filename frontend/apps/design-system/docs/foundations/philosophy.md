# 设计哲学

Nerv-IIP 设计系统服务于**数字工厂控制平面**——一个被工人、班组长、工艺与运维每天长时间盯着的生产系统。它的美学不是装饰，而是为了让人**更快、更准、更少出错**地完成生产动作。以下原则按优先级排列，冲突时上位者胜。

## 1. 产品优先

> 产品 / UX / PM 视角 **>** 业务与行业标准 **>** 工程实现便利。

先问"对使用的人最好的形态是什么"，再考虑业务习惯与行业惯例，最后才让工程可行性介入。**不要被现成组件限制**——当既有组件不足以表达正确的产品形态时，大胆造新件，而不是把体验迁就到组件能力上。

设计与评审时使用**真实感的种子数据**（真实工单号、工序、设备、节拍），而不是 `Lorem ipsum` 或 `占位文本`——只有看到接近真实的密度与边界，才能判断设计是否成立。

## 2. 引导，而非说明书

界面靠**结构、层级、状态与微交互**来引导操作，而不是靠成段的解说文字。

- 不要把页面写成"使用手册"。能用布局和视觉表达的，就不要用整段文字解释。
- 展示**真实可读的编号**——`WO-2406-0421`、`WC-03`——而不是 `工单编号占位` 这类提示。编号本身就是信息。
- 空状态、错误、加载都是设计的一部分，用引导性的下一步动作代替"暂无数据"。

## 3. 原版零改动，定制靠复制重建

底层 shadcn-vue / reka-ui 的**原版组件一律不改**。需要定制时，**复制一份重建**为我们的 Pro / Mobile 组件，再叠加设计令牌。

- 永远通过 `@nerv-iip/ui` / `@nerv-iip/ui-mobile` 的稳定导出边界使用组件，**不深 import** shadcn 内部文件。
- 新组件命名不与 `@nerv-iip/ui` 既有导出冲突。
- 这样原版可随上游升级，我们的定制层独立演进，互不污染。

## 4. 黑主题 · 动态色 · 亮暗自适应

视觉基准是**深色**（工厂环境、长时间注视更护眼），同时完整支持亮色。品牌强调色是**运行时可切换的动态色**（写入 `--brand`），一套语义令牌驱动全部表面，详见[设计令牌](/foundations/tokens)与[色彩](/foundations/color)。

## 5. 克制的工艺感

动效与质感服务于反馈与连续性，不喧宾夺主：

- 过渡用统一的缓动令牌（`--ease-out-quart / expo / in-out-quart`）。
- 拖拽类交互要有**跟手 + 橡皮筋阻尼 + 回弹**的真实手感（侧滑、抽屉、下拉刷新共用同一条阻尼曲线）。
- 覆盖层（弹窗 / 抽屉 / 菜单 / Tooltip）带克制的玻璃质感（半透明 + backdrop-blur），不过分。

## 6. 三表面一致，但各自原生

桌面 PC、车间一体机、移动 PDA 共享**同一套令牌与设计语言**，但各自遵循平台的原生手感与触控基线——移动端有安全区、≥44px 触控目标、手势；一体机大屏大卡、减少操作路径。详见[三个表面](/guide/surfaces)。
