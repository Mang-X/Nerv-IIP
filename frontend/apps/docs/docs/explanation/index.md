# 概念解释

概念解释回答"业务为什么这样运作"：单据之间如何流转、状态为什么这样设计、跨域协作在哪里
发生。它不教操作，读完应该能看懂系统里发生的事。

## 当前内容

- [核心业务流程图](/processes/)：六张流程图概览产品主线——工程资料、计划生产、仓储库存、
  质量审批、设备维护、条码追溯。每个节点标注角色、页面入口、对象状态与当前缺口。

## 规划中

以下主题随功能批次同步补齐：

- 单据与状态机总览：核心业务对象的状态流转与允许的操作。
- 组织、环境与权限模型：数据为什么按组织/环境隔离，页面入口为什么与权限相关。
- 跨域协作解释：计划、执行、库存、质量、设备之间的事实如何互相驱动。
- ERP 采购/销售/财务流程图：补齐第七、八张主线图。
