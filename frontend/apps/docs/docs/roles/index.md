# 按角色入门

不确定该看哪篇文档时，从这里开始：先找到你的角色，角色页会列出"第一周要会的路径"。
每条路径都是一串页面操作，以一个业务结果收尾；路径表同时如实标注当前能不能走通。

## 状态口径

- ✅ **可用**：业务结果当前可以在系统里达成。
- 🟡 **部分可用**：结果可以达成，但关键子能力缺失，标注处直接引用对应 GitHub issue。
- ⛔ **缺口**：结果当前达不成，标注处直接引用对应 GitHub issue。

状态跟随产品演进按季度盘点更新；缺口闭环后路径状态改回"可用"。

## 为什么路径表里有缺口标注

产品和文档同步演进。走查中发现"路走不通"时，不会把它写成已实现，而是标注缺口并直接
引用跟踪它的 GitHub issue——你可以据此判断该等待、绕行，还是反馈优先级。
