# 质检员：第一周路径

质检员把检验与不合格处置走成闭环。第一周先走通下面 7 条路径，每条路径都以一个业务结果收尾。
状态口径见[按角色入门](/roles/)。

## 第一周路径

| #   | 路径（页面操作串）                                                          | 业务结果                               | 状态                                                                                                                                                                |
| --- | --------------------------------------------------------------------------- | -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | 在 `/quality/inspections` 执行检验并记录结果                                | 形成通过/不通过的检验记录              | ✅ 可用                                                                                                                                                             |
| 2   | 在 `/quality/inspection-tasks` 按来源和时限处理待检任务，进入检验后提交结果 | 待检任务不遗漏，并在提交检验记录后闭合 | ✅ 可用；支持从 WMS 收货单按真实来源单号定位对应待检任务                                                                                                            |
| 3   | 检验不合格时在 `/quality/ncrs` 开 NCR                                       | 不合格品进入受控处置                   | ✅ 可用                                                                                                                                                             |
| 4   | 在 `/quality/ncrs` 提交处置，在 `/approval` 跟踪审批                        | 放行/返工/报废决定获得审批并关闭       | ✅ 可用                                                                                                                                                             |
| 5   | 对重复问题发起纠正预防与评审                                                | 不合格原因得到系统化跟踪               | 🟡 部分可用：NCR 详情的 CAPA/MRB 深化未交付（[#804](https://github.com/Mang-X/Nerv-IIP/issues/804)）                                                                |
| 6   | 在 `/quality/reason-codes` 维护原因码                                       | 检验与处置有统一原因口径               | ✅ 可用                                                                                                                                                             |
| 7   | 在 `/quality/analysis` 看质量分析，在 `/mes/traceability` 做批次追溯        | 质量趋势与问题批次可解释               | 🟡 部分可用：质量分析与 CAPA 读面、SPC 控制图未交付（[#677](https://github.com/Mang-X/Nerv-IIP/issues/677)、[#725](https://github.com/Mang-X/Nerv-IIP/issues/725)） |
