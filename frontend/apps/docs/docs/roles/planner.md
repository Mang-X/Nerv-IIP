# 计划员：第一周路径

计划员把需求变成可执行的生产计划。第一周先走通下面 5 条路径，每条路径都以一个业务结果收尾。
状态口径见[按角色入门](/roles/)。

## 第一周路径

| #   | 路径（页面操作串）                                          | 业务结果                       | 状态                                                                                                                                                    |
| --- | ----------------------------------------------------------- | ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | 在 `/planning` 维护需求并运行 MRP                           | 得到可处理的计划建议与 pegging | ✅ 可用                                                                                                                                                 |
| 2   | 在 `/planning` 接受计划建议，再到 `/mes/plans` 跟踪生产计划 | 建议进入 MES 可消费的生产计划  | ✅ 可用                                                                                                                                                 |
| 3   | 在 `/scheduling` 查看排产结果与产线负载                     | 拿到工单可参考的排程窗口       | 🟡 部分可用：后端与 facade 已支持锁定工序和持久化手动调整；甘特拖拽交互仍由 [#78](https://github.com/Mang-X/Nerv-IIP/issues/78) 跟踪 |
| 4   | 在 `/planning` 处理提前/延后/取消类异常建议                 | 计划收货与最新需求重新对齐     | 🟡 部分可用：MRP 例外消息面板未交付（[#805](https://github.com/Mang-X/Nerv-IIP/issues/805)）                                                            |
| 5   | 把采购类建议转成采购申请与采购订单                          | MRP 建议进入采购执行           | ⛔ 缺口：采购申请到采购订单的转化流程未打通（[#711](https://github.com/Mang-X/Nerv-IIP/issues/711)）                                                    |
